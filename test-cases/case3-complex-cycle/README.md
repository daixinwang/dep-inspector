# 测试用例 3：复杂循环
## 项目结构

```
src/
├── A.ts (imports B, C)
├── B.ts (imports C)
├── C.ts (imports D)
└── D.ts (imports A)
```

## 依赖关系

```
    ┌─────┐
    │  A  │
    └──┬──┘
       │
    ┌──┴──┐
    │     │
    ▼     ▼
  ┌───┐ ┌───┐
  │ B │ │ C │
  └─┬─┘ └─┬─┘
    │     │
    └──┬──┘
       ▼
     ┌───┐
     │ D │
     └─┬─┘
       │
       └──────► (back to A)
```

## 预期结果

- **节点数**：4
- **边数**：5
- **循环数**：1 或 2（取决于算法实现）
- **可能的循环路径**：
  - A → B → C → D → A
  - A → C → D → A

## 说明

这是一个更复杂的循环场景，其中：
- A 同时依赖 B 和 C
- B 也依赖 C
- C 依赖 D
- D 依赖 A，形成循环

这个结构中存在多条路径可以形成循环。
